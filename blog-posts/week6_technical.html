Week 6 Technical

Blocks/Procs/Lambdas in the Simplest Forms

Blocks

A block is code that is passed to a method using either curly braces, {...}, or "do...end" syntax. Curly braces are typically used for single line blocks and "do......end" syntax is usually used for multi-line blocks. Blocks are used commonly throughout ruby and are pretty routine in everyday uses.

Procs

Often we want to use a block of code more than once in a program and rather than retyping the block of code over and over again we can call on the block by using the Ruby Proc Object we can store a Proc into a variable and then pass it into any method that accepts a callable object. Procs also halt their methods and return the provided value when they encounter a "return" statement.

Lambdas

Lambda's are almost exaclty like Proc's with a couple of noticable differences. The first is that lambdas show the ArugmentErrors it receives and returns if they don't match. The second is that lambdas return their value to the method and allow it to continue.
